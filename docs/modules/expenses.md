# Expenses Module

> Standalone per-submitter voucher system with dual approval workflow.
> Introduced: **April 2026** · Migration **066** · Spec: `docs/superpowers/specs/2026-04-17-expenses-module-design.md`

---

## 1. Overview

Every employee (non-customer) can submit expense vouchers at `/expenses`. Vouchers get a unique `voucher_number` (e.g. `VIV-001`, `MAN-007`) auto-generated by a BEFORE INSERT trigger using the submitter's `voucher_prefix`.

**Two expense scopes:**
- **Project-linked** — `project_id` is set; 3-stage workflow (submitted → verified → approved).
- **General** — `project_id` IS NULL; 2-stage workflow (submitted → approved). No Verify stage.

---

## 2. Workflow (state machine)

### Project-linked (3 stages)
```
submitted → [PM/Founder verifies] → verified → [Founder approves] → approved
         → [PM/Founder rejects]  → rejected
```

### General (2 stages)
```
submitted → [Founder approves] → approved
          → [Founder rejects]  → rejected
```

**Revert** (Founder only): approved/rejected → earlier stage.

---

## 3. Tables

| Table | Purpose |
|-------|---------|
| `expenses` | Core voucher rows (renamed from `project_site_expenses`). `project_id` now nullable. |
| `expense_categories` | Master list of categories (travel, food, lodging, site_material, tools, consumables, labour_advance, miscellaneous). Founder + Finance can add/edit. |
| `expense_documents` | File references per expense (PDF/image). FK → `expenses.id` ON DELETE CASCADE. |
| `employees.voucher_prefix` | 3–5 letter unique prefix per active employee. Auto-derived from full_name on INSERT. |

**Status values:** `submitted` | `verified` | `approved` | `rejected`

**Constraint:** General expenses (project_id IS NULL) cannot have `verified_by` or `verified_at` set.

## 6. Past decisions

- **Renamed `project_site_expenses` → `expenses`** in migration 066 to make the module standalone. `project_id` made nullable to support general expenses.
- **Voucher prefix derived from full_name** (first 3–5 uppercase letters, unique per active employee). Trigger `trg_set_voucher_prefix` auto-runs on employee INSERT/UPDATE.
- **`get_expense_kpis` RPC** — aggregation in SQL (NEVER-DO #12 compliance). Returns 4 KPIs scoped to the caller's role.
- **Legacy rows backfilled** with `UNASSIGNED-###` sentinel voucher numbers (1164 rows at migration time).
- **`/vouchers` page** redirects to `/expenses?status=submitted`.

---

## 7. Gotchas

- **General expenses bypass Verify** — the DB CHECK constraint enforces this: `project_id IS NOT NULL OR (verified_by IS NULL AND verified_at IS NULL)`.
- **Voucher number is server-generated** — the Insert type requires `voucher_number: string` (not nullable) but the trigger fills it. Always pass `''` in Insert payloads; the trigger overwrites with the real value.
- **`expenses.status` column type is `text` (not an enum)** — validated by a CHECK constraint. The 4 values are `submitted | verified | approved | rejected`.
- **Document storage** — files go in Supabase Storage (`project-files` bucket). DB holds path strings only.
- **Finance can view categories admin** (`/expenses/categories`) but not approve/reject/verify expenses.
