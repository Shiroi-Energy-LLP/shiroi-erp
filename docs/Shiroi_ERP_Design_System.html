Component 10

Role-Based UI

Four roles in Shiroi ERP. Each sees a different version of the same data — not different apps, but different access and density levels.

👤
Customer

Views their own project dashboard, generation stats, invoices and O&M history only.

Read-only · No internal data · Simplified UI
💼
Sales

Manages leads, proposals, client communication and deal pipeline. Cannot modify technical specs.

Lead-to-project pipeline · No financials · No O&M
🔧
Engineer

Manages site surveys, installation records, O&M tickets, technical docs. Cannot delete records.

Full technical access · No pricing · No delete
⚙️
Admin

Full access. Can delete records, manage users, view all financials and configure system settings.

Badge pulse1500msease-in-outopacity (sync status only)

Motion Principles
Fast or invisible
UI motion should never delay the user. 150ms is the sweet spot — perceptible but not intrusive.
Purposeful only
Motion should communicate state change, not decorate. No spinning logos, no gratuitous bounces.
Exit faster than enter
Dismissals (modal close, toast exit) should be 100–150ms. Entry 200–300ms.
Reduced motion
Always respect prefers-reduced-motion. Disable all animations for users who need it.
